Discuss the scope of var, let and const

var has a scope that is limited to it's parent function which also includes the global function. So, although they are enclosed within their functions, they are not enclosed within blocks of code like 'if', 'while' or 'for' which can lead to issues due to their function-level scope. Furthermore, they get hoisted to their parent function but they are initialized with 'undefined'.

let and const have a scope which is block based which includes they are scoped within their respective blocks which can be 'if', 'while', 'for' or functions. The main difference between them is that const cannot be reassigned. Both of them get hoisted to the top of their block just like 'var' but they do not get initialized. So, accessing them before their declarations results in a `ReferenceError`.

Tell us the use cases of null and undefined

Use cases for null

  1. You can initialize a variable with 'null' to indicate that it has no value or it will be assigned one soon.
  2. In arrays, you can use null as a placeholder for empty slots, specifically for fixed-sized arrays.

Use cases for undefined

  1. You can initialize a variable with 'undefined' to indicate that it has no value or it will be assigned one soon. This happens by default, if nothing is assigned to a variable.
  2. You can use 'undefined' to check if a variable has been initialized or not. If a variable is still undefined, it is initialized but doesn't have a value assigned.
  3. You can check whether an object has a property or not. If it doesn't exist , it will return 'undefined' when accessed.
  4. You can check whether a function has been given all the arguments because when a function is called with fewer arguments than expected, the missing arguments are automatically assigned the value 'undefined'.
What do you mean by REST API?

It stands for Representational State Transfer Application Programming Interface. It is a set of guidelines and rules for designing networked applications that require communication between clients and servers. It relies on standard HTTP methods like GET, POST, PUT and DELETE. Furthermore, it emphasizes using URLs to represent resources. It is designed to be simple, scalable, and stateless.
